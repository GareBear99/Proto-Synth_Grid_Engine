# Contributing to Synth Grid Engine

Thanks for contributing.

## Before you start

Please use:

- **Issues** for bugs and scoped work
- **Discussions** for ideas, architecture, and open-ended direction
- **Pull Requests** for direct improvements

## Contribution style

This project values:

- deterministic behavior
- low-weight architecture
- modular structure
- readable code
- changes that preserve or improve runtime clarity

## Good contributions

- bug fixes with clear reproduction steps
- documentation improvements
- blueprint validation improvements
- module isolation and cleanup
- performance improvements that do not reduce clarity
- UI/HUD improvements that preserve readability

## Avoid

- huge unrelated refactors in a single PR
- changing engine direction without discussion first
- adding unnecessary dependencies
- hiding failures instead of surfacing them cleanly

## Pull request guidance

Keep PRs focused.

Include:

- what changed
- why it changed
- how it was tested
- screenshots if visuals changed

## Questions

Use Discussions for larger design conversations.
