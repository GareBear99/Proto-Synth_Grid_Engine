# Release Checklist

## Before release

- [ ] README is current
- [ ] Banner path works on GitHub
- [ ] Screenshots match current state
- [ ] Issue templates are present
- [ ] Discussions are enabled
- [ ] FUNDING.yml is updated with real handles
- [ ] Repo description is updated
- [ ] Topics are added
- [ ] Social preview image is uploaded

## Release notes should include

- major changes
- new blueprint or module support
- fixes
- known limitations
- screenshots or GIFs if visuals changed
